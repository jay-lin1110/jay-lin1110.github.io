NestJS 实现认证授权功能

Passport 是目前最流行的 Node.js 认证库 ，具有丰富的策略生态系统，可实施各种身份验证机制。

本地身份认证

Passport 提供了 Passport-local 的本地身份认证策略，它实现了一种用户名/密码身份验证机制。客户端先使用用户名和密码进行身份验证，一旦通过身份验证，服务器将发出 JWT，该 JWT 可以在后续请求的授权头中作为 token发送，以验证身份验证。

安装

npm i @nestjs/passport passport passport-local
npm i -D @types/passport-local

生成认证模块

# 生成认证模块
nest g mo auth
# 生成认证控制器
nest g co auth
# 生成认证服务
nest g s auth

认证控制器非必需，你可以根据需求将认证模块使用到用户、管理员等其他模块中。

本地身份认证服务

// auth.service.ts
import {
  Injectable
} from '@nestjs/common';
import { InjectModel } from 'nestjs-typegoose';
import { ReturnModelType } from '@typegoose/typegoose';
import { User } from '@app/mongo/models/user.model';

@Injectable()
export class AuthService {
  constructor(
    @InjectModel(User) private readonly userModel: ReturnModelType<typeof User>,
  ) { }
  async validateLocal(username: string, password: string): Promise<any> {
    const user = await this.userModel.findOne({ username });
    return (user && user.password === password) ? user : null;
  }
}

Tips：这里使用 @typegoose/typegoose 包注入用户模型提供用户服务，关于 MongoDB 模块的详细内容，请传送 MongoDB 章节。此外，用户密码属于密码数据，需要额外加密处理，可以通过散列或 md5 等加密方式加密，并通过 mongoose 的模型设计加密码字段设置为非必要读取字段，在需要密码的时候在取出密码字段。

实现本地身份认证策略

在 auth 文件夹下新建一个本地策略实现文件 local.strategy.ts

// local.strategy.ts
import { Strategy, IStrategyOptions } from 'passport-local';
import { PassportStrategy } from '@nestjs/passport';
import {
  Injectable,
  BadRequestException
} from '@nestjs/common';
import { AuthService } from '../auth.service'

@Injectable()
export class LocalStrategy extends PassportStrategy(Strategy, 'local-login') {
  constructor(private readonly authService: AuthService) {
    super({
      usernameField: 'username',
      passwordField: 'password',
    } as IStrategyOptions);
  }

  async validate(username: string, password: string): Promise<any> {
    const user = await this.authService.validateLocal(username, password);
    if (!user) {
      throw new BadRequestException('用户名或密码错误！');
    }
    return user;
  }
}

配置本地身份认证模块

在 auth.module.ts 中导入 passport 模块，提供本地身份认证策略。

// auth.module.ts
import { Module } from '@nestjs/common'
import { AuthController } from './auth.controller'
import { PassportModule } from '@nestjs/passport'
import { LocalStrategy } from './local.strategy'
import { AuthService } from './auth.service'

@Module({
  controllers: [AuthController],
  imports: [PassportModule],
  providers: [AuthService, LocalStrategy]
})
export class AuthModule {}

    }
  }

JWT 认证

安装

npm i @nestjs/jwt passport-jwt
npm i -D @types/passport-jwt

配置 JWT 密钥

官网学习文档建议我们将 JWT 密钥保存在配置文件中，我们可以结合环境变量和配置文件 .env 来配置 JWT 密钥。

# .env
# JWT 密钥
JWT_SECRET=这里是你配置的密钥

Tips：关于配置和环境变量的详细内容，请传送配置和环境变量章节。

库共享 JWT 模块

在库 common.module.ts 中配置 JWT 全局共享模块，结合环境变量使用 JWT 密钥

// libs/common.module.ts
import { Module, Global } from '@nestjs/common'
import { CommonService } from './common.service'
import { ConfigModule } from '@nestjs/config'
import { JwtModule } from '@nestjs/jwt'

@Global()
@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true
    }),
    JwtModule.registerAsync({
      useFactory: () => ({
        secret: process.env.JWT_SECRET
      })
    })
  ],
  providers: [CommonService],
  exports: [CommonService, JwtModule]
})
export class CommonModule {}

JWT 认证服务

// auth.service.ts
import {
  Injectable
} from '@nestjs/common';
import { InjectModel } from 'nestjs-typegoose';
import { ReturnModelType } from '@typegoose/typegoose';
import { JwtService } from '@nestjs/jwt';
import { User } from '@app/mongo/models/user.model';

@Injectable()
export class AuthService {
  constructor(
    @InjectModel(User) private readonly userModel: ReturnModelType<typeof User>,
    private readonly jwtService: JwtService,
  ) { }

  async login(user: any): Promise<any> {
    const payload = { id: user._id };
    return {
      token: this.jwtService.sign(payload),
    };
  }

  async verifyJwt(id: string): Promise<any> {
    const user = await this.userModel.findById(id);
    return user ? user : null;
  }
}

实现 JWT 认证策略

在 auth 文件夹下新建一个 JWT 策略实现文件 jwt.strategy.ts

// jwt.strategy.ts
import { Strategy, ExtractJwt, StrategyOptions } from 'passport-jwt';
import { PassportStrategy } from '@nestjs/passport';
import { Injectable } from '@nestjs/common';
import { AuthService } from '../auth.service'

@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor(private readonly authService: AuthService) {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      ignoreExpiration: false,
      secretOrKey: process.env.JWT_SECRET,
    } as StrategyOptions);
  }

  async validate(payload: any) {
    const user = await this.authService.verifyJwt(String(payload._id));
    return user;
  }
}

配置 JWT 认证策略

在 auth.module.ts 中导入 passport 模块，提供 JWT 认证策略。

// auth.module.ts
import { Module } from '@nestjs/common'
import { AuthController } from './auth.controller'
import { AuthService } from './auth.service'
import { PassportModule } from '@nestjs/passport'
import { LocalStrategy } from './local.strategy'
import { JwtStrategy } from './jwt.strategy'

@Module({
  controllers: [AuthController],
  imports: [PassportModule],
  providers: [AuthService, LocalStrategy, JwtStrategy],
  exports: [JwtStrategy]
})
export class AuthModule {}

构建授权访问路由守卫

用户登录成功后，使用 JWT 验证策略根据密钥和部分非敏感用户信息签署生成令牌 token，每次有效请求都必须携带该令牌，以此保护我们的接口。

// auth.controllers.ts
import {
  Controller,
  Get,
  Post,
  Body,
  UseGuards,
  Req,
  Injectable,
} from '@nestjs/common';
import { AuthDto } from './dto/auth.dto';
import { AuthGuard } from '@nestjs/passport';
import { AuthService } from './auth.service'
import { ApiBearerAuth } from '@nestjs/swagger';

@Injectable()
@Controller('auth')
export class AuthController {
  constructor(private readonly authService: AuthService) { }

  @Post('login')
  @UseGuards(AuthGuard('local-login'))
  async login(@Body() body: AuthDto, @Req() req) {
    return this.authService.login(req.user)
  }

  @ApiBearerAuth()
  @UseGuards(AuthGuard('jwt'))
  @Get('profile')
  profile(@Req() req) {
    const { username } = req.user;
    return {
      code: 0,
      message: 'success',
      username,
    };
  }
}

使用@ApiBearerAuth() 装饰器可以让我们测试接口的时候在请求头中携带 token 令牌，当然，你也可以在 main.ts 中初始化 swagger 时候进行配置。
