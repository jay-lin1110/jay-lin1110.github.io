微信小程序的网络请求

wx-request

在微信小程序原生开发 Mina 框架中，我们可以使用官方的网络请求接口wx-request()方法发送网络请求，该方法的基本使用类似于jQuery中的发送 Ajax 请求的ajax()方法，wx-request()方法接收一个网络请求配置对象，具体使用示例可参考官方文档链接wx-request

wx-request()方法使用示例

wx.request({
  url: 'test.php', //仅为示例，并非真实的接口地址
  data: {
    x: '',
    y: ''
  },
  header: {
    'content-type': 'application/json' // 默认值
  },
  success(res) {
    console.log(res.data)
  }
})

回调地狱

原生网络请求接口wx-request()方法并不支持Promise API风格的调用，这也意味着当我们处理存在继发关系网络请求时候很容易出现回调函数的多重嵌套，即当我们的网络请求存在必要的先后顺序，请求处理逐级依赖嵌套，容易形成回调地狱，这样使用十分不便利。

wx.request({
  url: ''http:localhost:3000/users'',
  data: {
    x: ""
  },
  success (data) {
    wx.request({
      url: http:localhost:3000/profiles',
      data: {
        x: data.id
      },
      success (res) {
        console.log(res.data)
      }
    })
  }
})

Promise 风格的 wx-request

方案一：自定义封装 Promise 风格的 wx-request

我们可以在原生网络请求接口wx-request()方法的基础上封装一个Promise风格的wx-request()请求方法，一般我们会在项目根目录下创建一个 utils 文件夹，用于存放公共的工具文件

具体实现

//  utils/wx-request.js
// 网络请求接口的基本地址，这是一个示例
// 更好的做法是将其单独放置在项目的配置文件中再导入进来
// 微信小程序官方支持类似于CommonJS的模块语法
// 你也可以使用ES2015的模块语法。小程序支持增强编译
const baseURL = 'http:localhost:3000'

// 自定义wx.request()的promise版本
// 你可以根据需要继续封装独立的get()或post()等方法
const wxRequest = options =>
  new Promise((success, fail) => {
    // 请求时候显示loading效果
    wx.showLoading({
      title: '拼命加载中=。=！',
      mask: true
    })

    const { url } = options
    options.url = `${baseURL}${url}`
    wx.request({
      ...options,
      success,
      fail,
      complete() {
        //   请求结束隐藏loading效果
        wx.hideLoading()
      }
    })
  })

export default wxRequest

使用示例

// 更好的做法是将API单独封装到一个api文件夹中
import wxRequest from 'utils/wx-request.js'
wx.request({
  url: '/users',
  data: {
    x: ''
  }
})
  .then(value => {
    return wx.request({
      url: 'profiles',
      data: {
        x: value.id
      }
    })
  })
  .then(value => {
    console.log(value.data)
  })

方案二：使用 miniprogram-api-promise 包 Promise 化 API

使用方法

  1. 安装

    npm i -S miniprogram-api-promise

  2. 使用

    // 只需要在微信小程序入口文件app.js中调用一次promisifyAll
// 你可以根据需求进一步封装，例如加入loading效果，单独抽离api文件夹等

// 将wxp挂载在全局或直接导出
const wxp = {}
// promise化所有的wx-api接口
promisifyAll(wx, wxp)
wxp.request().then()

// promise化单个的wx-api接口
promisify(wx.request)().then()

方案三：使用 Flyio

Flyio是一个类似于 axiso 的轻量且强大的 ajax 请求库，和 axios 一样，支持多种 JavaScript 运行环境 ，目前 Fly.js 支持的平台包括：浏览器 、Nodejs、Weex、React Native 和 Quick App 外，还同样兼容微信小程序，所以我们可以借助它来像使用 axios 一样在微信小程序中发送网络请求，它基于 Promise 所以天生就支持 Promise 风格 API，所以我们不用再额外处理，只需要根据具体业务逻辑进一步使用。

使用方法

  1. 下载安装

    npm i -S flyio

  2. 使用

    var Fly=require("../lib/wx") //wx.js为您下载的源码文件
// var Fly=require("flyio/dist/npm/wx") //npm引入方式
var fly=new Fly(); //创建fly实例

//添加拦截器
fly.interceptors.request.use((config,promise)=>{
    //给所有请求添加自定义header
    config.headers["X-Tag"]="flyio";
    return config;
})
//配置请求基地址
fly.config.baseURL="https://wendux.github.io/"
...

Page({
  //事件处理函数
  bindViewTap: function() {
    //调用
    fly.get("http://10.10.180.81/doris/1/1.0.0/user/login",{xx:6}).then((d)=>{
      //输出请求数据
      console.log(d.data)
      //输出响应头
      console.log(d.header)
    }).catch(err=>{
      console.log(err.status,err.message)
    })
    ...
  })
})
